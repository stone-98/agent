## 注意
目前的实现方案不兼容多进程的程序，因为目前的实现方案采用监听主进程的状态，当主进程挂了，那么Agent就会认为该程序死了，所以就会出现如下问题：

例如使用Agent管理Nginx，配置完成directory、command、isAutoStart等参数，Agent会将Nginx程序自动拉起，但是因为Nginx的实现架构采用多进程的方式，主进程负责管理工作进程以及相关配置文件的管理等任务，而工作进程才是真正提供Web服务，那么当主进程意外挂掉后，工作进程其实还在运行，主进程并不影响提供Web服务，那么Agent将会根据isAutoRestart参数的设置对Nginx进行启动，如果启动那么此时会出现旧的工作进程还在运行，这是不符合逻辑的。

关于这一点，暂时没有想到解决方案。
